    am

    @mixin am($module, $trait) { ... }

    Description

    Attribute Modules for CSS

    Parameters

    parameterNameparameterDescriptionparameterTypeparameterDefault value
    $module—noneString—none
    $trait—noneString—none

    Content

    This mixin allows extra content to be passed (through the @content directive).

    Example

    Usage

    @include am(module) {
   color: red;
 }

 @include am(module, blue) {
   color: blue;
 }

 @include am(module, large) {
   font-size: 2em;
 }

    CSS Output

    [am-module] {
  color: red;
}

[am-module~="blue"] {
  color: blue;
}

[am-module~="large"] {
  font-size: 2em;
}

    position

    @mixin position($values, $sides, $pos, $auto-overwrite) { ... }

    Description

    Generate position properties in shorthand form.

    Parameters

    parameterNameparameterDescriptionparameterTypeparameterDefault value
    $values

    List of numbers or auto

    		List—none
    $sides

    List of sides (top, left, bottom, right or all)

    		List—none
    $pos

    Position keyword or none

    		String—none
    $auto-overwrite

    If true all non specified $sides will be set to auto

    NOTE: $values get mapped to $sides in the order they are declared, see usage below

    https:gist.github.com/nicolas-cusan/8fc0cef0f466f6ca2ebc

    		Bool—none

    Example

    Usage

    .my-class {
  @include position(0, top left);
}

    CSS Output

    .my-class {
  position: fixed;
  top: 0;
  left: 0;
  right: 0;
  bottom: 0;
}

    Usage

    .my-class {
  @include position(auto 200px 3em, top left right);
}

    CSS Output

    .my-class {
  position: absolute;
  top: auto;
  left: 200px;
  right: 3em;
}

    Usage

    .my-class {
  @include position(0 3px, left top, none);
}

    CSS Output

    .my-class {
  top: 3px;
  left: 0;
}

    Usage

    .my-class {
  @include position(0 200px, top left, $auto-overwrite: true);
}

    CSS Output

    .my-class {
  position: absolute;
  top: 0;
  left: 200px;
  right: auto;
  bottom: auto;
}

    border-radius

    @mixin border-radius() { ... }

    Description

    Round all corners by a specific amount, defaults to value of config-get("i-border-radius").

    When two values are passed, the first is the horizontal radius and the second is the vertical radius.

    Note: webkit does not support shorthand syntax for several corners at once. So in the case where you pass several values only the first will be passed to webkit.

    Parameters

    None.

    Example

    Usage

    .simple   {
  @include border-radius(4px, 4px);
}
.compound {
  @include border-radius(2px 5px, 3px 6px);
}
.crazy    {
  @include border-radius(1px 3px 5px 7px, 2px 4px 6px 8px)
}

    CSS Output

     .simple {
   -moz-border-radius: 4px / 4px;
   -webkit-border-radius: 4px 4px;
   border-radius: 4px / 4px;
 }
 .compound {
   -moz-border-radius: 2px 5px / 3px 6px;
   -webkit-border-radius: 2px 3px;
   border-radius: 2px 5px / 3px 6px;
 }
.crazy {
  -moz-border-radius: 1px 3px 5px 7px / 2px 4px 6px 8px;
  -webkit-border-radius: 1px 2px;
  border-radius: 1px 3px 5px 7px / 2px 4px 6px 8px;
}
